# Security Policy

## Supported Versions

Security fixes are handled on the default branch, `main`. The project does not
currently maintain separate long-term support release branches.

## Reporting A Vulnerability

Please do not open a public issue for a suspected vulnerability.

Use GitHub's private vulnerability reporting or security advisory workflow for
this repository when available. If that option is not available in your GitHub
interface, open a minimal public issue asking for a private contact channel and
do not include exploit details, secrets, private data, or sensitive logs.

Useful reports include:

- affected commit, version, or installation method;
- operating system and Python version;
- a minimal reproduction when safe to share;
- expected impact;
- whether the issue may expose credentials, private data, or arbitrary code
  execution.

## Disclosure

Maintainers will acknowledge valid reports as soon as practical, investigate the
impact, and coordinate a fix before public disclosure when the issue is security
sensitive.
